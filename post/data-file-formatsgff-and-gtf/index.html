GFF

GFF (General Feature Format)，用来描述基因组特征的文件，我们现在主要使用的gff3 version。

GFF文件是由tab键分割的，具有九列的文件。

description
1. Seqid 序列的名称(chromosone or scaffold)。
2. source 注释的来源，未知用.代替，一般指的是产生此gff3文件的软件方法。
3.feature type 类型，与gtf相比，相对自由，建议使用sequence ontology的惯例名称。
4. start 开始位点，从1开始计数（bed文件从0开始）。
5. end 终止位点。
6. score 得分，对于一些可量化的属性，在此设置一个数值以表示程度的不同。如果为空，可以用.代替。
7. strand “＋”表示正链，“－”表示负链，“.”表示不需要指定正负链。
8. phase 对于编码蛋白质的CDS来说，本列指定下一个密码子开始的位置。可以是0、1或2，表示到达下一个密码子需要跳过的碱基个数。
9. attributes 一个包含众多属性的列表，格式为“标签＝值”（tag=value），不同属性之间用;相隔。ID: gff中序列的唯一标识，用于属性分类。Name: 指定属性的名称。展示给用户的就是该属性。Alias: 名称的代称或其他，当存在其他名称时使用该属性。Note: 描述性的一些说明。

GTF

GTF (Gene Transfer Format)，主要用于对基因的注释，我们现在主要使用的GTF2 version。
1. Seqid 代表序列的ID，一般为染色体的ID
2. source 可以是数据库的名称，也可以是预测软件的名称，也可以为空用.填充，eg.Refseq数据库，GeneScan软件。
3. feature type 有gene、transcript、exon、5’UTR、CDS、start_codon、stop_codon、3‘UTR
4. start 区间的起始位置
5. end 区间的终止位置
6. score 软件提供了统计值，如果没有，就用.填充
7. strand 代表正负链的信息, +表示正链，-表示负链，?表示不清楚正负链的信息，当正负链信息没有意义时，可以用.填充
8. phase 当描述的是CDS区间信息时，需要指定翻译时开始的位置，取值范围有0,1,2两种
9. attributes 每种属性写法为key value, 注意和gff中key=value有所区别，而且必须有gene_id和transcript_id这两个属性， 多个属性用分号分隔。

Non-coding tanspcript: gene，transcipt，exon

protein-coding transcipt：gene，transcipt，exon，5’UTR，CDS，start_codon，stop_codon，3‘UTR

attributes

1

gene_id "Em:U62317.C22.6.mRNA"; transcript_id "Em:U62317.C22.6.mRNA"; exon_number 1

  • gene

    • gene_id：Ensembl数据库中的基因ID以ENSG作为前缀
    • gene_version：基因ID的版本号，用于区分不同版本
    • gene_name：的是基因的symbol, 和NCBI数据库中的gene_symbol一致
    • gene_source：来源
    • gene_biotype：基因类型

  • transcript

    • gene的基础上，增加了以下
    • transcript_id
    • transcript_version
    • transcript_name
    • transcript_source
    • transcript_biotype

  • exon

    • exon_number
    • exon_id
    • exon_version

GFF vs GTF

  • GFF全称为general feature format，这种格式主要是用来注释基因组。
  • GTF全称为gene transfer format，主要是用来对基因进行注释。
  • GTF前8个字段与GFF基本相同，第9个字段已扩展为属性列表。每个属性由一个类型/值对组成。属性必须以分号结束，并且与后面的任何属性之间只隔一个空格。

目前两种文件可以方便的相互转化:使用gffread

1

gffread  YL.gff -T -o li.gtf
