一、Docker的介绍

1.1 为什么要用docker

  1. 解决环境不一致的问题。
  2. 多用户操作系统下，保证不相互影响。
  3. 解决运维成本过高。
  4. 解决安装成本过高的问题。

1.2 docker的由来

1.3 docker的思想

  1. 集装箱：将所有需要的内容放到集装箱中，谁需要就派大海疼到中央仓库拿就行。
  2. 标准化的运输方式
    • 运输的标准化：docker有一个码头，所有的集装箱都在这个码头上，当需要某个环境，就直接派大海疼去搬运这个集装箱就行。
    • 命令的标准化：docker提供了一系列的名利来帮助我们去获取集装箱等操作。
    • 提供了REST的API：提供图形化的界面，Rancher
  3. 隔离性：docker在运行集装箱的时候，会在linux内核中单独开辟一片空间，这篇空间不影响其他程序的运行。
    • 注册中心：超级码头，其上有许多码头
    • 镜像：images集装箱
    • 容器：container运行起来的镜像

#常用版本
docker run -d -p 宿主机端口:容器端口 --name 容器名称 容器id
# -d：后台运行
# -p 宿主机端口:容器端口：为了映射linux的端口，从win找到linux,再从linux到容器
# --name：指定容器名称
# -i：保持容器运行，通常与 -t（terminal）同时使用。加入 it 这俩个参数后，容器创建后自动进入容器中，退出容器后，容器自动关闭
# -t：为容器重新分配一个伪输入终端，通常与 -i 同时使用
# -d：以守护（后台）模式运行容器。创建一个容器在后台运行，需要使用 docker exec 进入容器。退出后，容器不会关闭
# -it：创建的容器一般称为交互式容器， -id 创建的容器一般称为守护式容器
# --name：为创建的容器命名,已经用 --name 创建过的不能再创建
# -v：docker run -i -t -v /software/:/mnt/software/ e934aafc2206 /bin/bash
假设我们的所有安装程序（jdk，tomcat）都放在了mac的/software/目录下，现在需要将其挂载到容器的/mnt/software/目录下。
#交互式容器：容器创建后立马进入容器内部，默认bin/bash
docker run -it --name 镜像名字 镜像id
#守护式容器
docker run -id --name 设置容器名字 镜像id

  1. 容器数据的持久化，通过挂在实现。宿主机与容器目录之间共享文件。

  2. 不同容器间实现容器数据交换。

    • 通过将不同的容器挂在同一目录下实现。

    • 数据卷容器

      1
2
3
4

      #创建启动c3数据卷容器，使用 -v 参数 设置数据卷
docker run –it --name=c3 –v /volume centos:7 /bin/bash
#创建启动 c1 c2 容器，使用 --volume-from 参数 设置数据卷
docker run –it --name=c1 --volumes-from c3 centos:7 /bin/bash
docker run –it --name=c2 --volumes-from c3 centos:7 /bin/bash
#c1 c2 c3 有一个文件夹 volume
