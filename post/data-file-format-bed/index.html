bed

BED (Browser Extensible Data)，注释文件，可以用Genome Browser可以进行可视化。bed每一行有3个必须的字段和其他9个可选的字段。

  • 必须的3列
  1. chrom：染色体或scaffold名称
  2. chromStart：feature在染色体或者scaffold的起始位置，初始位置为0。
  3. chromEnd：feature在染色体或scaffold的终止位置。
  • 可选的9列
  1. name： 定义BED行的名称。当轨道打开到完全显示模式时，此标签显示在Genome浏览器窗口中BED行的左侧，或者在打包模式下直接显示在项目的左侧。
  2. score：得分在0到1000之间。如果此注释数据集的轨迹线useScore属性设置为1，则得分值将确定显示此要素的灰度级别（较高的数字=较深的灰色）。此表显示 Genome Browser将BED分数值转换为灰色阴影。
  3. strand："." (=no strand) or “+” or “-”
  4. thickStart：绘制特征的起始位置，例如基因的起始密码子。若没有，则设置为chromstart的值
  5. thickEnd：绘制特征的终止位置，如基因的终止密码子
  6. itemRgb：R，G，B形式的RGB值（例如255,0,0）。如果轨道行 itemRgb属性设置为“On”，则此RBG值将确定此BED行中包含的数据的显示颜色。注意：建议使用此属性的简单颜色方案（八种颜色或更少颜色），以避免压倒Genome浏览器和Internet浏览器的颜色资源。
  7. blockCount：the number of blocks(exon) in the bed line
  8. blockSizes：大小的逗号分隔列表。此列表中的项目数应与blockCount相对应。
  9. blockStarts：逗号分隔的块开始列表。应该相对于chromStart计算所有 blockStart位置。此列表中的项目数应与blockCount相对应

以gencode最为权威，Ensemble、NCBI 、UCSC也有提供下载。

处理工具有bedtools等。
