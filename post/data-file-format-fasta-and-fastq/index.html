fasta

fasta是最基本的序列（核苷酸或氨基酸）信息储存格式。文件以.fa或.fasatq为后缀。

主要分为两个部分：

  • 定义行：单行，以> 开头，后面紧跟的是这条序列的名称，具有唯一性。
  • 序列信息部分：所有的序列（碱基或氨基酸）都可以放在一行储存，也可以放在多行储存，建议单行的长度不超过80个字符，这样方便阅读。碱基的信息可能存在大小写不一致的情形，蛋白质序列信息中，*表示蛋白质翻译结束。

fastq

fastq同样是以文本信息储存序列信息的格式。文件以.fa或.fastq后缀。

fasta是经过人为排列（头信息和序列信息）的序列文件，而fastq则储存的是测序仪产生的原始数据，由测序的图像数据转换而来。与fasta不相同的是，它除了存储序列本身外还存储了序列中每个单元所对应的质量分数，所以fastq格式通常用于高通量测试数据的存储。

fastq文件中一般由4行内容组成，每4行为一个独立的单元，也叫read。

以+开头，一般什么信息也没有，在旧版的FASTQ文件中会直接重复第一行的信息，但现在一般什么也不加（节省存储空间）。

  • 第四行

是测序read的质量值Q，与测序的碱基的信息一一对应，表示每个碱基可靠程度，用ASCII码表示。

如果测得准，则其出错的概率就越低，质量值就越高。

Q = -10*log(p_error) #p_error: 测序错误率

测序平台 ASCII码范围 下限 质量值类型 质量值范围
sanger/Illumina 33-126 33 Phred quality socre 0-93

Eg. ASCII码33对应的字符串!表示质量值为0。

ASCII码值-33=Q

如’J' 对应ASCII 码为74，而如果质量体系为Phred33，则通过减去33，则可知’J’对应的质量值为41。
